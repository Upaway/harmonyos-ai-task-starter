# HarmonyOS 发布证书申请指南

## 📝 概述

本文档详细说明如何为 JustStart 应用申请 HarmonyOS 正式发布证书，以便发布到华为应用市场。

**应用信息：**
- 应用包名：`com.juststart.app`
- 应用名称：JustStart
- 当前版本：1.0.0 (versionCode: 1000000)

---

## 🚀 完整流程

### 第一步：生成 CSR 文件

1. **运行 CSR 生成脚本**
   ```bash
   ./generate-csr.sh
   ```

   脚本会在 `cert/` 目录下生成：
   - `release_private_key.pem` - 私钥文件（⚠️ 请妥善保管）
   - `release.csr` - 证书签名请求文件

2. **验证生成结果**
   ```bash
   ls -la cert/
   ```

---

### 第二步：在华为开发者联盟申请证书

#### 2.1 登录开发者联盟
1. 访问：https://developer.huawei.com/consumer/cn/
2. 使用您的开发者账号登录
3. 完成实名认证（如未完成）

#### 2.2 创建应用（如未创建）
1. 进入「AppGallery Connect」
2. 点击「我的应用」→「创建」
3. 填写应用信息：
   - 应用名称：JustStart
   - 应用包名：com.juststart.app
   - 应用类型：选择「应用」

#### 2.3 申请发布证书
1. 进入「AppGallery Connect」
2. 点击「用户与访问」→「证书管理」
3. 点击「申请证书」
4. 填写证书信息：
   - **证书类型**：发布证书
   - **证书名称**：JustStart_Release_Cert
   - **选择应用**：JustStart (com.juststart.app)
   - **上传 CSR 文件**：选择 `cert/release.csr`

5. 提交申请并等待审核

#### 2.4 下载证书文件
审核通过后，下载以下文件：
- **发布证书**：`.cer` 或 `.p7b` 格式
- **Profile 文件**：`.p7b` 格式

---

### 第三步：配置发布签名

#### 3.1 创建发布签名配置文件

将下载的证书文件放到项目的 `cert/` 目录：
```
JustStart/
├── cert/
│   ├── release_private_key.pem          # 私钥文件
│   ├── release.csr                       # CSR 文件
│   ├── JustStart_release.cer             # 发布证书（下载得到）
│   └── JustStart_profile.p7b             # Profile 文件（下载得到）
```

#### 3.2 转换证书格式（如需要）

如果下载的是 `.cer` 格式，需要转换为 `.p12` 格式：

```bash
# 转换为 P12 格式
openssl pkcs12 -export \
  -in cert/JustStart_release.cer \
  -inkey cert/release_private_key.pem \
  -out cert/release_keystore.p12 \
  -name "releaseKey"
```

#### 3.3 更新 build-profile.json5

在 `build-profile.json5` 中添加发布签名配置：

```json5
{
  "app": {
    "signingConfigs": [
      {
        "name": "default",
        "type": "HarmonyOS",
        "material": {
          "certpath": "cert/JustStart_release.cer",
          "keyAlias": "releaseKey",
          "keyPassword": "您的密钥密码",
          "profile": "cert/JustStart_profile.p7b",
          "signAlg": "SHA256withECDSA",
          "storeFile": "cert/release_keystore.p12",
          "storePassword": "您的存储密码"
        }
      }
    ],
    "products": [
      {
        "name": "default",
        "signingConfig": "default",
        "compatibleSdkVersion": "6.0.2(22)",
        "targetSdkVersion": "6.0.2(22)",
        "runtimeOS": "HarmonyOS"
      }
    ]
  }
}
```

---

### 第四步：构建发布版本

#### 4.1 构建发布版 HAP/APP 包

在 DevEco Studio 中：
1. 点击菜单 `Build` → `Build Hap(s)/APP(s)` → `Build APP(s)`
2. 等待构建完成
3. 生成的发布包位于：`entry/build/default/outputs/default/`

#### 4.2 验证签名

```bash
# 验证 APP 包签名
hap verify-app --in-file entry/build/default/outputs/default/entry-default-signed.app
```

---

### 第五步：上传到应用市场

#### 5.1 准备上传材料
- **应用包**：`.app` 或 `.hap` 文件
- **应用图标**：512x512 PNG 格式
- **应用截图**：至少 3 张，建议尺寸 1080x1920
- **应用描述**：功能介绍、更新说明等
- **隐私政策**：应用隐私政策链接

#### 5.2 上传发布
1. 进入 AppGallery Connect
2. 选择应用 → 「分发」→「版本信息」
3. 上传应用包并填写版本信息
4. 提交审核

## ⚠️ 重要注意事项

### 安全提醒
1. **私钥保管**：`release_private_key.pem` 文件包含私钥，请妥善保管，不要泄露或上传到代码仓库
2. **证书备份**：下载的证书文件请做好备份，丢失后需要重新申请
3. **密码管理**：签名密码请妥善保管，建议使用密码管理工具

### 证书有效期
- **发布证书**：通常有效期为 1-3 年
- **Profile 文件**：通常有效期为 1 年
- 过期后需要重新申请或续期

### 常见问题

**Q: CSR 文件生成失败？**
A: 请确保系统已安装 OpenSSL。macOS 通常已预装，可通过 `openssl version` 验证。

**Q: 证书申请被拒绝？**
A: 常见原因：
- 应用信息填写不完整
- CSR 文件格式错误
- 应用包名与已发布应用冲突
- 开发者资质审核未通过

**Q: 签名失败？**
A: 检查：
- 证书文件路径是否正确
- 密码是否正确
- 证书是否过期
- Profile 文件是否匹配应用包名
