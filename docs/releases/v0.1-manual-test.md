# V0.1 真机手动测试计划

## 测试结论口径

- P0 全部通过：可以进入下一轮小范围使用测试。
- P1 有失败：先修复再扩大测试范围。
- P2 有失败：记录问题，不阻塞 V0.1 主流程验证。

## 环境检查（P0）

- DevEco Studio 可正常打开工程
- 已安装与工程配置匹配的 HarmonyOS NEXT SDK（当前验证基线：6.0.2）
- 真机系统版本与工程目标版本兼容（当前验证基线：HarmonyOS 6.0.2）
- 真机已开启开发者模式和 USB 调试
- DevEco Device Manager 能识别真机
- 工程可完成 Debug 构建并安装到真机
- 首次启动无闪退，后台日志无连续崩溃

## 主流程（P0）

1. 打开应用，首页显示任务输入、时长选择、今日摘要。
2. 不输入任务时，“开始做事”按钮不可点击。
3. 输入合法任务后，默认时长为 25 或本地保存的默认值。
4. 切换 25 / 50 分钟后进入确认页，任务和时长显示正确。
5. 在确认页点击“返回修改”可回首页，输入内容仍在当前页面栈可见。
6. 点击“正式开始”进入专注页，倒计时开始。
7. 点击“暂停”后时间停止，再点“继续”后恢复。
8. 点击“提前完成”先进入确认，再落到结果页，结束原因正确。
9. 点击“放弃”先进入确认，再落到结果页，结束原因正确。
10. 等待倒计时结束自动跳到结果页，结束原因显示为正常结束。

## 结果与存储（P0）

1. 在结果页输入备注并保存。
2. 返回首页后，今日次数和今日分钟数有更新。
3. 进入统计设置页，摘要数据与首页一致。
4. 统计设置页最近记录列表展示本次任务。
5. 杀掉应用进程后重新打开，今日摘要和最近记录仍存在。
6. 设备重启后重新打开，已保存数据仍存在。

## 设置（P1）

1. 在统计设置页切换默认专注时长并保存。
2. 返回首页后，默认时长与设置保持一致。
3. 点击提醒占位和 AI 占位按钮后，状态提示变化，重新进入页面后仍保持上次值。

## 导航与异常路径（P1）

1. 从确认页返回首页，任务输入和时长保持页面栈内状态。
2. 从专注页点击提前完成后取消，仍停留在专注页且计时继续。
3. 从专注页点击放弃后取消，仍停留在专注页且计时继续。
4. 结果页保存按钮连续快速点击，只生成一条记录。
5. 统计设置页点击返回，回到进入前的首页。

## 前后台与锁屏（P1）

1. 专注计时中切到后台 30 秒再返回，确认应用不闪退，页面状态仍可操作。
2. 专注计时中锁屏 30 秒再解锁，确认应用不闪退，页面状态仍可操作。
3. 专注计时中接入/断开充电器，确认页面布局和计时无明显异常。

## 输入边界（P1）

1. 输入 50 个中文字符，可以进入确认页。
2. 尝试输入超过 50 个字符，输入框限制生效。
3. 结果备注输入 100 个字符，可以保存。
4. 尝试输入超过 100 个字符，输入框限制生效。
5. 输入前后带空格的任务，确认页展示去除首尾空格后的任务。

## UI 适配（P2）

1. 默认字体大小下，所有页面无文字遮挡和按钮溢出。
2. 系统字体调大一档后，首页、专注页、结果页、统计页仍可完成主流程。
3. 亮屏旋转或折叠状态变化时，应用不闪退。

## 空状态（P2）

1. 清空本地存储后重新打开应用。
2. 首页摘要显示 0。
3. 统计设置页显示自然空状态文案。

## 本轮不验收

- 正式签名和应用市场提审。
- 提醒通知的真实触达能力。
- AI 功能真实调用能力。
