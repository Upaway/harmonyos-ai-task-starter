# v0.1.1 - Open-source maintenance update

开源仓库的**小步维护版本**：文档补充与叙事对齐。**不包含**新的核心业务功能，也**不是**应用商店意义上的功能更新。

## What changed

- **CHANGELOG**：将先前累积的开源维护项归入 **v0.1.1**，便于与 Git 标签、GitHub Release 对齐。
- **故障排查文档**：新增 [`docs/troubleshooting.md`](../troubleshooting.md)，覆盖 DevEco 打开工程、SDK 版本、`build-profile`、签名、真机、同步失败及勿提交敏感材料等常见问题。
- **README**：补充指向 troubleshooting 的链接；在 Roadmap 摘要附近明确 **v0.2 / v0.4** 分工（避免读者误以为 AI 拆解已进入当前里程碑）。
- **Quickstart**：增加指向 troubleshooting 的入口。

## Documentation updates

- `docs/quickstart.md`：引导读者在卡顿时查阅 `docs/troubleshooting.md`。
- `README.md`：文档索引表增加 troubleshooting 与本文链接。

## Open-source maintenance

- 版本叙事：`CHANGELOG` 中 **[Unreleased]** 空置，已完成维护记入 **[0.1.1]**。
- 仍遵循：**MIT**、`CONTRIBUTING`/`SECURITY`、模板化 `build-profile.json5.template`，真实签名仅本地。

## What did not change

- **应用逻辑**：专注闭环、页面流程、本地持久化行为未改。
- **UI**：无改动。
- **依赖**：未新增 ohpm/npm 等依赖。
- **权限与包名**：`bundleName`、应用显示名、权限声明未改。
- **云同步 / 账号系统**：未引入。
- **生产环境 AI API**：未接入；产品内的 AI 相关仍为占位或路线图范围。

## Known limitations

- 仍为早期基准工程：体验与文档会陆续迭代。
- 排障文档依赖 DevEco / SDK 版本演进，若与你的 IDE 界面不一致，请以华为官方文档为准。
- Git **标签 `v0.1.1`** 由维护者在合并本批次变更后**手动**创建（本仓库自动化脚本不会代打 tag）。

## Next steps

- 维护者可按需创建 **`v0.1.1`** tag，并在 GitHub Release 中粘贴或节选本文。
- 继续用小 PR 迭代文档与 Issue 闭环（截图 GIF、上架 checklist 等单独跟踪）。
